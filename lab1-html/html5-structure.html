HTML5 Semantic Structure

This page demonstrates the use of semantic HTML5 elements to organise content meaningfully.

What are semantic elements?

Semantic elements clearly describe their meaning to both the browser and the developer. Examples include: <header>, <nav>, <main>, <article>, <aside>, and <footer>.

Why use semantic HTML?

Using semantic elements makes your code easier to read, improves accessibility for screen readers, and helps search engines understand your content better.

Tip: Always prefer semantic elements over generic <div> tags when the element has a clear role on the page.
